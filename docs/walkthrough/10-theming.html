Chapter 10

Theming & the Design System

Eight runtime themes, from one elegant trick: every colour is a CSS variable in two forms, so both Tailwind utilities and their opacity modifiers re-theme instantly. Adding a theme is one token block plus one registry line — nothing else changes.

The dual-variable token trick

In styles/tokens.css, every colour token exists twice:

--iris-500: #b5532f;          // hex — for inline styles & SVG fills: var(--iris-500)
--iris-500-rgb: 181 83 47;    // "r g b" channels — for Tailwind

Tailwind's config maps colours to rgb(var(--iris-500-rgb) / <alpha-value>). That single line is why bg-iris-500 and bg-iris-500/20 both re-theme — the opacity modifier composes with the channel form. A theme is one :root[data-theme="id"] block that overrides these variables; switching themes is just setting data-theme on <html>. Every class and inline var() across the app cascades from there.

Decision · variables, not a rebuildThemes change at runtime with zero JS recomputation and no flash — the saved theme is applied before first paint in main.tsx. The alternative (Tailwind's dark: variants or multiple built stylesheets) couldn't express five light + three dark palettes cleanly, nor re-theme inline SVG. One source of truth, cascading.

The eight themes

LightDark
Terracotta (default), Mango Sunshine, Sage Grove, Lavender, Rose QuartzIndigo Night, Espresso (warm), Forest Night (green)

Dark themes set color-scheme: dark, invert the parchment/ink ramps, and lift the status colours (rose/sage) for legibility. A token --on-primary is the text colour that sits on a filled accent (near-white on dark accents, dark ink on bright ones like Mango's gold) so buttons always read.

Area colours are never themedThis is a rule: the Bloom's petals use the area's own colour regardless of theme, so the bloom always means the same thing. Themes only change the light around your data.

Cross-device theme sync

The choice is stored per-device in localStorage (applied before paint) and mirrored to the synced settings row so it follows you across devices. The cloud save is debounced (500ms): rapid toggling would otherwise spam the row, and its out-of-order realtime echoes would flicker the selection. Debounce means one save of the settled choice — one echo, equal to what's showing, no flicker.

The area colour palette & picker

36 colours across nine hue families (Greens, Teals, Blues, Purples, Pinks, Reds, Oranges, Yellows, Neutrals). The ColorPicker lays them out in a responsive auto-fill grid (fills landscape width, compact height) in hue order, so contrasting choices sit a glance apart — and it marks colours already used by other areas, so your wheel stays legibly varied instead of collapsing into similar shades.

App-native controls (no OS chrome)

A design goal: the app's experience filters down to every click — no dropping to jarring OS pickers. So the native controls were all replaced with themed, touch-first components:

  • TimeField — a popover time picker (hour / minute / AM·PM), replacing <input type="time">.
  • DateField — a month-grid calendar popover, replacing <input type="date">.
  • SegmentedControl — pill toggles for few choices (time of day).
  • SelectMenu / SortMenu — app dropdowns for many choices (cadence, Home sort), with descriptions.

The "Remind me at" field was also made an obvious tappable affordance (it used to read as empty space).

Motion & the calm boot

  • Framer Motion throughout, wrapped in MotionConfig reducedMotion="user" so prefers-reduced-motion makes every spring instant app-wide.
  • Watercolour wash — a soft accent gradient behind sheets and the ritual player.
  • One boot screen. A static logo cover in index.html (before React), the same logo in the React splash, and the same in the update overlay — so update → reload → boot reads as a single screen that fades to reveal the app. The "Updating Harmony" label only appears on an actual version bump; ordinary reopens show just the logo and land straight on Home with no floor/wait.
  • List reflow: Home's habit list uses layout animation so a card glides into place when the sort order changes (e.g. tapping one done under "Still to do").
