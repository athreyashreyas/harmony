Chapter 03

The Data Model

Six shapes carry the whole app: Area, Habit, Log, plus NotificationSettings (which also holds Rituals), NudgeHistory, and the local Outbox. They live in packages/shared/src/types.ts so client and worker agree exactly.

Area

A part of life you name.

interface Area {
  id, userId
  name           // your word: "Body", "Mind" (≤ 30 chars)
  color          // hex from a curated palette; NEVER themed
  importance     // 'core' | 'matters' | 'optional'
  whySentence    // your reason, brought back to you (≤ 240)
  order          // priority order (drag to reorder)
  createdAt, archivedAt
  driftSensitivity?   // 'low' | 'default' | 'high'
  reminderTimeOfDay?  // when this area's nudges arrive
}
Two dials per areaImportance shapes how readily the Bloom petal fills (ch. 04) and whether drift nudges fire (optional areas never nag). Drift sensitivity is a second, independent multiplier on how quickly "quiet" is called (ch. 07).

Habit

A small practice inside an area. The polarity field is the fork between a tend and a tug.

interface Habit {
  id, userId, areaId
  name
  cadence        // the repeat rule (see below)
  timeOfDay      // 'morning'|'afternoon'|'evening'|'anytime' — sort/label only
  reminderTime   // "HH:mm" or null
  startDate, endDate
  order
  color?         // override, else inherits the area's colour
  polarity?      // 'tend' (default) | 'ease' (a tug)
  tugWeight?     // how much one logged tug sets you back (ease only)
  weight?        // share of the area's Bloom (default 1 = equal)
}

Cadence — the repeat rule (a discriminated union)

kindMeaning
dailyEvery day.
weekdays / weekendsMon–Fri / Sat–Sun.
specific-days: { days:[0..6] }Certain weekdays (0=Sun).
times-per-week: { times }Flexible: shown every day, you pick which.
every-n-days: { n }Counted from the start date.
every-n-weeks: { n }Same weekday every n weeks (fortnightly, etc.).
every-n-months: { n }Same day-of-month, clamped to month length (monthly, quarterly, yearly).
Why times-per-week shows every dayRather than guess which days you'll do a "3× a week" habit, Harmony shows it daily and lets the day's own log mark it done. It respects that flexible habits are flexible — the Bloom just needs the count, not a prescribed schedule.

Log

One tap = one Log. This is the highest-volume, ever-growing table, which is why it gets special sync treatment (ch. 06).

interface Log {
  id, userId, habitId, areaId
  date       // "YYYY-MM-DD" local — the day it counts for
  loggedAt   // timestamp it was tapped (drives time-of-day insights)
  note       // optional "how did that feel?" (≤ 1000)
}

A compound index [habitId+date] makes "is this habit logged today?" an instant lookup. On the server, two extra columns were added for incremental sync: updated_at (bumped by a trigger) and deleted_at (soft-delete). More in chapter 06.

Note the two timestamps: date is the day the log is for (you can log a past day); loggedAt is when you actually tapped. Insights' "time of day" rhythm uses loggedAt; the calendar uses date.

NotificationSettings — the synced per-account row

One row per user. Beyond notifications it has become the home for small synced preferences, because it's already a synced singleton.

interface NotificationSettings {
  masterEnabled, mutedAreaIds[], dndStart, dndEnd
  habitReminders, dailySummary
  theme?           // chosen theme id, follows you across devices
  lastSeenVersion? // "What's new" shown once per account
  homeSort?        // Home list order preference
  rituals?         // Ritual[] — named habit-stacking flows
}
interface Ritual { id, name, habitIds[] }
Decision · Rituals as JSON on settingsA ritual is small per-account data (a name + an ordered list of habit ids). Rather than a whole new synced table (migration, sync mappings, realtime, outbox), it rides the existing settings row as a jsonb column. It syncs across devices for free via the settings path. Trade-off: no per-ritual realtime granularity — acceptable for something you edit rarely.

NudgeHistory & the Outbox

  • NudgeHistory — a record of every nudge sent (drift, reminder, summary), with the composed text, channel (in-app/push), and time. It powers anti-spam (don't repeat a template for the same area within 3 days) and the "already shown today" guards.
  • Outbox (local only, Dexie) — the durable write queue: { op: 'upsert'|'delete', table, payload, onConflict }, auto-incrementing id so writes drain in the order they were made (which also respects FK order: area before habit before log).
